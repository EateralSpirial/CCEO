# Plan 3

## 本期目标
增强安全、审计和长期运行能力，使总经理成为可以稳定托管多个 Codex 项目的本地运营系统。

## 本期范围
- 审计日志与敏感配置保护。
- 多项目策略与角色隔离。
- 自动化健康检查与恢复。
- 更细的 cron 操作和风险防护。

## 前置依赖与准备项
- term_1 和 term_2 已提供完整对象模型和基本执行能力。

## 详细任务列表（按顺序）
1. 建立敏感配置脱敏和展示策略。
2. 实现统一审计日志与操作回溯页。
3. 加入项目级权限边界和风险提示。
4. 增加 cron / Codex / Qdrant 健康检查与告警。
5. 完成总经理运行快照的导出与备份。

## 交付物清单
- 审计与安全中心。
- 健康检查与告警面板。
- 备份导出能力。

## 验收标准
- 每类关键操作都有审计记录。
- 敏感字段不会明文暴露到 UI。
- 健康检查能覆盖 Codex、Qdrant、cron-loop 关键链路。

## 风险与回滚策略
- 审计与权限增强不得破坏已有项目数据。
- 健康检查默认只读，不引入破坏性自恢复。

## 与后续期衔接
- 为多设备、多用户或更复杂渠道接入留出升级路径。
